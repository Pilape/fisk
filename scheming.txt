== COMPROMISES ==
- How do we make sure the language is tiny? What does tiny even mean in this context?
    - Small library
        - A few KiB in size
        - No stdlib

    - Simple language
        - Few keywords
        - Simple syntax
        - Few types

- ANSI C for portability?
    - No, not worth it
    - Not that hard to convert later

- Will quotations be lists or atomic?
    - Atomic
        - Easier to implement
        - Less of that dynamic memory management

    - List
        - Adds a list structure in the process
        - More versatility

- How will lists work?
    - Dynamic/automatic
        - Easier to use
        - Needs some kind of allocation

    - Static/manual
        - Easier to implement
        - Kinda annoying to use

- Anything resembling a struct?
    - Probably at some point. Idk when or what tho.

== PREFIXES ==

' Character
$ Variable
@ External function/primitive (With the sole exception of single char primitives)
